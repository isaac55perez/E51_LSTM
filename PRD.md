# PRD — LSTM Frequency Extractor (E51_LSTM)

## 1. Purpose
Implement the M.Sc. assignment "Developing an LSTM System for Frequency
Extraction from a Mixed Signal" (Dr. Yoram Segal, Nov 2025). Given a noisy
mixed sinusoidal signal `S[t]` and a one-hot selector `C`, the system must
output the clean target sinusoid `Target_i[t] = sin(2π f_i t)` for the
selected frequency `f_i ∈ {1, 3, 5, 7} Hz`.

## 2. Stakeholders
- **Student (author)** — implements, trains, reports.
- **Instructor (Dr. Segal)** — evaluates adherence to §2–§5 of the PDF.

## 3. Functional requirements

### 3.1 Data generation (PDF §2 with documented interpretation)
- `fs = 1000 Hz`, duration `10 s` → `10 000` samples per frequency.
- For each `f_i ∈ {1,3,5,7}`:
  - `A_i ~ U(0.8, 1.2)` and `φ_i ~ U(0, 2π)` drawn **once per realisation**.
  - `ε_i(t) ~ N(0, σ²)` (σ = 0.1) additive Gaussian noise per timestep.
  - `noisy_i(t) = A_i · sin(2π f_i t + φ_i) + ε_i(t)`.
- Mixed signal `S(t) = ¼ Σ_i noisy_i(t)`.
- Clean targets `Target_i(t) = sin(2π f_i t)`.
- Dataset composition: 4 selector blocks × 10 000 rows = 40 000 rows of
  `(S[t], one_hot(4), Target_i[t])` as per PDF Table 2.
- Single seed (`numpy.random.default_rng(1)`) for reproducibility.

**Why not per-timestep A_i(t) / φ_i(t)?** A literal read of PDF §2.2 implies
per-timestep random phase, which reduces `S[t]` to an i.i.d. noise stream
(zero information about `t` or `f_i`) and forces any model to the mean
predictor. This PRD uses the standard noisy-sinusoid interpretation instead
— see `ARCHITECTURE.md` for a full discussion.

### 3.2 Train/test split (user override)
- 80/20 temporal split **per selector block**.
  - Train: first 8 000 timesteps of each selector (32 000 rows total).
  - Test:  last 2 000 timesteps of each selector (8 000 rows total).
- Splitting by time (not random shuffling) keeps the stateful LSTM evaluation
  meaningful.

### 3.3 Model
- PyTorch `nn.LSTM` with `input_size=5`, `hidden_size=128`, `num_layers=1`,
  `batch_first=True`, followed by `nn.Linear(128, 1)`.
- Forward returns both the per-timestep prediction and the `(h, c)` state so
  the caller can thread state across batches.

### 3.4 Training protocol
- Sequence-length sweep: `L ∈ {1, 3, 7, 15, 30}`.
- Per epoch: stream the 4-selector batch through non-overlapping windows of
  length `L`; initialise state at the start of the stream, then carry across
  windows, detaching to truncate BPTT.
- Loss = MSE; Optimiser = Adam, `lr = 1e-3`; up to 60 epochs with patience 8.
- Test evaluation warms the hidden state through the 8 000-step training
  block before predicting on the test block.
- Checkpoint on best **test** MSE.

### 3.5 Memory-reset policy (§4.3 alternative)
| Situation                        | Reset? | Reason |
|----------------------------------|:-----: |--------|
| Between timesteps, same selector |  No    | Stateful temporal learning. |
| Between L-length windows         |  No (detach only) | Truncated BPTT. |
| Between selectors / epochs       |  Yes   | New conditioning context. |
| Train → test transition          |  Yes   | Evaluate from a fresh state. |

### 3.6 Metrics & artefacts
- Per epoch: train MSE, test MSE (logged to `metrics.json`).
- Per experiment: `summary.json` with best epoch + runtime.
- Per-L plots (`curves.png`, `triple_compare_f{i}.png`,
  `pred_vs_target_f{i}.png`, `all_frequencies.png`).
- Aggregate plot (`output/comparison_L_sweep.png`) and `docs/RESULTS.md`.

### 3.7 Graphs (PDF §5.2)
- **Graph 1** — noisy `S`, clean target, LSTM output on one axis, per `f_i`.
- **Graph 2** — 4-panel plot, one sub-plot per frequency.
- Both generated by `save_sample_outputs.py`.

## 4. Non-functional requirements
- Reproducibility: running `run_training.py` twice on the same machine
  yields byte-identical dataset caches (same seed).
- Runtime: full sweep ≤ 15 minutes on CPU (target: a mid-range laptop).
- No GPU required; auto-detect `cuda` if available.

## 5. Acceptance criteria
1. `python run_training.py` completes all 5 L values without errors.
2. Each `output/L_*/best_model.pt`, `metrics.json`, `summary.json`,
   `curves.png` exists.
3. `python save_sample_outputs.py` produces the PDF §5.2 graphs; the LSTM
   output visibly tracks the clean green sinusoid, not the gray noisy trace.
4. `python analyze_results.py` writes `docs/RESULTS.md` with the 5-row table
   and the comparison plot exists at `output/comparison_L_sweep.png`.
5. Best-L generalisation ratio `test_mse / train_mse ≤ ~1.5` on this seed.
